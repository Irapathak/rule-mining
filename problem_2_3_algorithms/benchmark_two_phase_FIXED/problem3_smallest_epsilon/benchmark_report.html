🎯 Find Smallest Epsilon - Benchmark Results

Two-Phase Search Algorithm (Exponential + Binary)

30 Experiments | 10 Rules | 3 Delta Values | Runtime: 432.75 minutes

💡 Algorithm Overview

Phase 1: Exponential Search - Quickly find upper bound by doubling epsilon

Phase 2: Binary Search - Refine to find exact smallest epsilon

largest ε (heterogeneous) = smallest ε (homogeneous) - 1

Above ε*, the rule is homogeneous. At ε* - 1, violations exist.

📖 Metrics Glossary - How Each Column is Calculated

Delta (δ) Fixed Input Parameter. The minimum subgroup size threshold. All candidate subgroups must contain at least δ individuals to be evaluated.
Input: User-specified value
Smallest ε (Homogeneous) Algorithm Output. The smallest epsilon threshold where the rule becomes homogeneous (no subgroups violate the threshold). This is the primary result we're searching for.
Found via: Exponential search + Binary search
Largest ε (Heterogeneous) Derived Metric. The largest epsilon value where violations still exist. Due to monotonicity, this is exactly one less than the smallest homogeneous epsilon.
Calculation: smallest_ε_homogeneous - 1
Oracle Calls Efficiency Metric. Total number of times we invoked the homogeneity oracle (FPGrowth algorithm) to check if a rule is homogeneous at a given epsilon. Lower is better; indicates search efficiency.
Count: Total calls to FPGrowth oracle during binary search (plus one upper-bound check)
Runtime (seconds) Performance Metric. Wall-clock time (in seconds) to complete the entire binary search for this specific rule and delta combination (ε ∈ [0, ε_max]).
Measurement: end_time - start_time
Dataset Size Context Information. The total number of rows (individuals) in the dataset for this particular rule. Larger datasets may require more computation per oracle call.
Value: len(dataframe)
Rule ID Identifier. Unique numerical identifier for each treatment-condition pair being tested.
Value: 1, 2, 3, ...
